SEARCHLISTINGS API ENHANCEMENT - PAGINATION AND SORTING
========================================================

OVERVIEW
--------
The searchListings endpoint has been enhanced to support pagination and sorting while maintaining full backward compatibility with existing frontend implementations. All search functionality remains the same, with added pagination and sorting capabilities.

ENDPOINT
--------
GET /listings/search

QUERY PARAMETERS
----------------

SEARCH PARAMETERS (Existing - unchanged):
-----------------------------------------
1. keyword (string, optional)
   - Searches in title and description fields (case insensitive)
   - Example: keyword=laptop

2. mainCategory (string, optional)
   - Filter by main category
   - Example: mainCategory=Electronics

3. subCategory (string, optional)
   - Filter by sub category
   - Example: subCategory=Computers

4. subSubCategory (string, optional)
   - Filter by sub-sub category
   - Example: subSubCategory=Laptops

5. minPrice (number, optional)
   - Minimum price filter
   - Example: minPrice=100

6. maxPrice (number, optional)
   - Maximum price filter
   - Example: maxPrice=1000

NEW PAGINATION & SORTING PARAMETERS:
------------------------------------
7. page (integer, default: 1)
   - Specifies which page of results to return
   - Only used when 'limit' is also specified
   - Example: page=2

8. limit (integer, default: 0)
   - Number of listings per page
   - When set to 0 or omitted, returns ALL results (backward compatible)
   - When set to a positive number, enables pagination
   - Example: limit=10

9. sortBy (string, default: 'recent')
   - Determines the sorting order of search results
   - Case insensitive (recent, RECENT, Recent all work)
   - Available options:
     * 'recent' - Sort by newest first (created date descending)
     * 'older' - Sort by oldest first (created date ascending)
     * 'priceAsc' - Sort by price ascending (lowest to highest)
     * 'priceDesc' - Sort by price descending (highest to lowest)
   - Invalid values default to 'recent' sorting
   - Secondary sort criteria applied for consistency
   - Example: sortBy=priceAsc

RESPONSE FORMATS
----------------

1. WITHOUT PAGINATION (Backward Compatible):
   When no 'limit' is specified or limit=0, the response is exactly the same as before:
   
   Response: Array of listing objects
   [
     {
       "_id": "...",
       "title": "...",
       "description": "...",
       "price": 100,
       "createdBy": { "name": "John Doe" },
       ...
     },
     ...
   ]

2. WITH PAGINATION:
   When 'limit' is specified (> 0), the response includes pagination metadata and search criteria:
   
   Response: Object with listings, pagination info, and search criteria
   {
     "listings": [
       {
         "_id": "...",
         "title": "...",
         "description": "...",
         "price": 100,
         "createdBy": { "name": "John Doe" },
         ...
       },
       ...
     ],
     "pagination": {
       "currentPage": 1,
       "totalPages": 5,
       "totalCount": 50,
       "hasNextPage": true,
       "hasPrevPage": false,
       "limit": 10
     },
     "sortBy": "recent",
     "searchCriteria": {
       "keyword": "laptop",
       "mainCategory": "Electronics",
       "subCategory": null,
       "subSubCategory": null,
       "minPrice": 100,
       "maxPrice": 1000
     }
   }

USAGE EXAMPLES
--------------

1. Legacy search (still works):
   GET /listings/search?keyword=laptop
   - Returns all laptop listings sorted by newest first
   - Response format unchanged

2. Search with pagination:
   GET /listings/search?keyword=laptop&page=1&limit=10
   - Returns first 10 laptop listings with pagination metadata

3. Search with sorting:
   GET /listings/search?keyword=laptop&sortBy=priceAsc
   - Returns all laptop listings sorted by price (lowest first)
   - No pagination metadata in response

4. Complex search with pagination and sorting:
   GET /listings/search?keyword=laptop&mainCategory=Electronics&minPrice=500&maxPrice=2000&page=2&limit=5&sortBy=priceDesc
   - Returns second page of 5 laptop listings in Electronics category, price range 500-2000, sorted by price (highest first)

5. Category search with pagination:
   GET /listings/search?mainCategory=Electronics&subCategory=Computers&page=1&limit=20&sortBy=recent
   - Returns first 20 computer listings sorted by newest first

6. Price range search:
   GET /listings/search?minPrice=100&maxPrice=500&sortBy=priceAsc&limit=15
   - Returns first 15 listings in price range 100-500, sorted by price ascending

7. Case insensitive sorting:
   GET /listings/search?keyword=phone&sortBy=PRICEDESC&limit=10
   - Works with uppercase sorting parameters

8. Error examples:
   GET /listings/search?keyword=laptop&page=0          → 400 error: "Page number must be greater than 0"
   GET /listings/search?keyword=laptop&limit=-5        → 400 error: "Limit must be non-negative"
   GET /listings/search?keyword=laptop&sortBy=invalid  → Works fine, defaults to 'recent' sorting

TECHNICAL DETAILS
-----------------
- All listings with state 'sold' are automatically filtered out
- Search uses case-insensitive regex for keyword matching in title and description
- Price filters are converted to numbers automatically
- Default sorting is by creation date (newest first) for consistency
- Pagination metadata is only included when limit > 0
- Total count is calculated efficiently using MongoDB's countDocuments with search query
- The API uses skip() and limit() for efficient pagination

SORTING IMPROVEMENTS
--------------------
- Case insensitive sorting parameters (recent, RECENT, Recent all work)
- Secondary sort criteria for consistent ordering:
  * Date sorts use _id as tiebreaker for same timestamps
  * Price sorts use creation date as tiebreaker for same prices
- Input validation for page (must be > 0) and limit (must be >= 0)
- Graceful handling of invalid sortBy values (defaults to 'recent')
- Sort criteria mapping:
  * recent → { created: -1, _id: -1 }
  * older → { created: 1, _id: 1 }
  * priceAsc → { price: 1, created: -1 }
  * priceDesc → { price: -1, created: -1 }
